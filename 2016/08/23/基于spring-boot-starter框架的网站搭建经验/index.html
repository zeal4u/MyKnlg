基于spring-boot-starter框架的网站搭建经验

发表于 2016-08-23   |   分类于 Web

本文只是在看了spring官方文档后，搭建网站过程中的一些小小经验

  • 1.由于spring-boot-starter将tomcat整合在一起，使得整个工程可以像一般的jar工程一样可以运行，在配置主类Application时，注意设置
    @ComponentScan(basePackages = {"com.XXX"})
    即所有组件所在的基包名。如果Application也在这层包内，则可以省略。
    要注意的一点是，尽量避免使用default包，否则可能会产生莫名的错误。这在构建工程的时候也是一个不好的习惯。

  • 2.配置文件在spring-boot-starter框架中几乎被消灭掉了，大部分的配置都采用注解形式，但是在进行数据库配置时，
    最好还是创建一个名为application.properties的配置文件，或者是一个yml文件，文件名是spring默认的，如果不喜欢这个名字可以更改，具体方法参见spring docs，在此不多说了。
